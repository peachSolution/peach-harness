# 배포 구조

> 작성일: 2026-03-14
> 의사결정: 단일 플러그인 + 자체 마켓플레이스 방식

## 결정 사항

peach-harness 저장소는 **단일 플러그인 + 자체 마켓플레이스**이다. `.claude-plugin/` 안에 marketplace.json과 plugin.json이 공존한다.

## 배경

Claude Code의 `/plugin install`은 등록된 마켓플레이스에서 플러그인을 검색하는 명령이다.
마켓플레이스 없이는 `/plugin install`로 설치할 수 없다.

저장소 구조 (planning-with-files 등 실제 작동하는 플러그인과 동일 패턴):
- `.claude-plugin/marketplace.json` — 마켓플레이스 정의 (`"source": "./"`)
- `.claude-plugin/plugin.json` — 플러그인 정의 (컴포넌트)

**주의: marketplace.json에 skills 배열을 넣으면 plugin.json과 충돌한다.**
marketplace.json은 source 경로만 지정하고, 컴포넌트 정의는 plugin.json에 위임한다.

## 플러그인 vs 마켓플레이스 (개념 정리)

| 구분 | 플러그인(앱) | 마켓플레이스(앱스토어) |
|------|------------|-------------------|
| 저장소 구성 | 플러그인 1개 = 저장소 1개 | 플러그인 여러 개 = 저장소 1개 |
| 필요 파일 | `plugin.json` | `marketplace.json` + 각 플러그인의 `plugin.json` |
| 설치 방법 | 마켓플레이스 등록 필요 | `/plugin marketplace add owner/repo` → 안에서 선택 설치 |
| 자동 업데이트 | 지원됨 | 지원됨 |

### 마켓플레이스 확장 (향후)

여러 플러그인을 만들 경우, 별도 마켓플레이스 저장소를 만든다.

```
peach-marketplace/                    ← 별도 저장소
├── .claude-plugin/marketplace.json
└── plugins/
    ├── peach-harness-plugin/
    ├── peach-analytics-plugin/
    └── peach-deploy-plugin/
```

## 현재 배포 구조

```
peach-harness/
├── .claude-plugin/
│   ├── marketplace.json             ← 마켓플레이스 정의 (source: "./")
│   └── plugin.json                  ← 플러그인 정의
├── .claude/
│   └── skills/                      ← 로컬 전용 스킬 (배포 제외)
│       └── peach-release/           ← peach-harness 전용 릴리스 스킬
├── skills/                          ← 배포 스킬 (skills.sh + 플러그인 공통)
├── agents/
├── hooks/
├── templates/
└── docs/
```

### 내부 전용 스킬 (.claude/skills/)

`.claude/skills/`에 위치한 스킬은 SKILL.md frontmatter에 `metadata.internal: true`를 설정하여 **skills.sh 배포에서 제외**한다.

| 스킬 | 이유 |
|------|------|
| `peach-release` | peach-harness 저장소 릴리스 전용. 외부 프로젝트에서 불필요. |

**배포 제외 메커니즘 (`metadata.internal: true`):**
- skills CLI는 `.claude/skills/`도 `PRIORITY_PREFIXES`로 탐색하므로 디렉토리 위치만으로는 제외되지 않는다.
- SKILL.md frontmatter에 `metadata.internal: true`를 설정하면 `npx skills add`에서 자동 제외된다.
- `--skill '*'`이나 `--skill peach-release`로 명시적 지정하면 internal 스킬도 설치된다.
- 환경변수 `INSTALL_INTERNAL_SKILLS=1`로도 강제 포함 가능 (개발용).
- Claude Code 로컬 로딩에는 영향 없다 (플러그인/로컬 스킬 시스템은 skills CLI와 별도 경로).

## 설치 방법

### 권장 설치 조합

- **Claude Code만 사용**: Claude Code 플러그인 방식 설치를 권장한다.
- **Claude Code + 다른 AI 함께 사용**: Claude Code는 플러그인으로 설치하고, Codex / Cursor / Gemini / Antigravity 등 다른 AI에는 `skills.sh`로 별도 설치한다.
- **비-Claude AI만 사용**: `skills.sh`로 설치한다.

> **중요**: Claude Code 플러그인으로 설치한 내용은 Cursor, Codex, Gemini 등 다른 AI가 직접 로드하지 못한다.
> 다른 AI에서도 사용하려면 해당 AI 대상에 `skills.sh` 방식으로 별도 설치해야 한다.
> 기본 권장 시나리오는 "Claude는 플러그인, 나머지는 `skills.sh`"이며, 전체 AI를 `skills.sh` 하나로 통일하는 방식은 기본 권장하지 않는다.

### Claude Code 플러그인 (권장)

agents/, hooks/, MCP 서버 등 Claude Code 전용 기능을 포함한다.

```bash
# 1. 마켓플레이스 등록
/plugin marketplace add peachSolution/peach-harness

# 2. 플러그인 설치
/plugin install peach
```

### skills.sh 설치 (비-Claude AI 권장 / Claude에서는 대안)

SKILL.md 오픈 스탠다드 기반으로 Claude Code, Cursor, Codex CLI, Antigravity, Copilot, Gemini CLI 등을 지원한다.
설치와 업데이트 모두 동일 명령이다. `-y` 플래그로 기존 설치를 덮어쓴다.

```bash
npx skills add peachSolution/peach-harness \
  -a codex \
  -a cursor \
  -a gemini-cli \
  -a antigravity \
  -g -y
```

> **`--skill '*'` 비권장**: `--skill '*'`은 내부 전용 스킬(`metadata.internal: true`)까지 포함한다.
> `--skill` 옵션 없이 실행하면 배포용 스킬만 자동 설치된다.

### 대안: 전체 AI를 skills.sh로 통일 설치

전체 AI를 `skills.sh` 방식으로 설치해도 동작은 한다.
다만 Claude Code에서는 플러그인 방식이 기본 권장이다.

```bash
npx skills add peachSolution/peach-harness \
  -a claude-code \
  -a codex \
  -a cursor \
  -a gemini-cli \
  -a antigravity \
  -g -y
```

## 슬래시 커맨드 정책

> 결정일: 2026-04-05
> 의사결정: commands/ 미사용, skills/ 단일 체계

### 결정 사항

skills.sh와 플러그인 양쪽 지원을 위해 `commands/` 폴더를 사용하지 않는다.
사용자 호출은 `skills/*/SKILL.md`의 슬래시 명령으로 통일한다.

| 설치 방식 | 슬래시 형식 | 예시 |
|----------|-----------|------|
| skills.sh | `/peach-gen-ui` | 하이픈 구분 |
| 플러그인 | `/peach:peach-gen-ui` | Tab 자동완성 시 `{플러그인명}:{스킬명}` 형식으로 표시됨 |

> **플러그인 네임스페이스 중복 표시**: Claude Code 플러그인은 슬래시 자동완성 시 `{플러그인명}:{스킬명}` 형식으로 표시한다.
> `peach` 플러그인의 `peach-db-query` 스킬은 `peach:peach-db-query`로 표시된다.
> `peach-` prefix는 skills.sh 설치 시 네임스페이스 충돌 방지를 위해 필수이므로 현상 유지한다.
> `codex:codex-rescue` 등 공식 플러그인도 동일한 패턴을 사용한다.

### 배경: commands/ 폐기 이유

Claude Code 플러그인은 `commands/*.md` 파일로 `/peach:gen-ui` 형태의 네임스페이스 커맨드를 지원한다.
그러나 skills.sh는 `commands/`를 인식하지 못해 skills.sh 사용자에게는 커맨드가 노출되지 않는다.

둘 다 유지하면 같은 스킬에 대해 `/peach-gen-ui`(skills.sh)와 `/peach:gen-ui`(commands/)가 중복 노출된다.
이중 관리 부담과 사용자 혼란을 피하기 위해 commands/를 폐기하고 skills/ 단일 체계로 통일한다.

### user-invocable 옵션 동작 (검증 완료)

SKILL.md frontmatter의 `user-invocable` 필드로 슬래시 명령 노출을 제어할 수 있다.

| 설정 | 슬래시 자동완성 | AI 내부 호출 |
|------|--------------|------------|
| `user-invocable: true` 또는 생략 | **노출됨** | 가능 |
| `user-invocable: false` | **숨겨짐** | 가능 |

**검증 방법** (2026-04-05):
1. `~/.agents/skills/`에 테스트 스킬 2개 생성 (true/false 각 1개)
2. `~/.claude/skills/`에 심볼릭 링크 배치
3. 새 세션에서 `/test-` 입력 → `true`만 자동완성에 노출, `false`는 숨겨짐 확인

**활용**: 내부 전용 스킬(AI가 Skill 도구로만 호출)은 `user-invocable: false`로 설정하여 슬래시 목록 오염을 방지한다.

## 업데이트

- GitHub에 push하면 설치한 사용자가 업데이트 가능
- `/plugin` → Installed 탭 → 해당 플러그인 선택 → Update
- "Enable auto update" 설정 시 Claude Code 실행마다 자동 최신화
