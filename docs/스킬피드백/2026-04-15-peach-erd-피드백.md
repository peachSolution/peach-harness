## 2. 해결 방법 / 우회 전략

### 문제 #1: preview 후 PNG/SVG git 변경 자동 처리 부재

**원인**: 현재 스킬은 미리보기 + 파일 저장까지만 수행하고 종료됨.
PNG/SVG는 렌더링 메타데이터 차이로 항상 git diff에 잡힘.

**올바른 판단 기준**:
- `erd.mmd`가 **변경됨** → PNG/SVG도 함께 커밋 대상
- `erd.mmd`가 **미변경** (preview만 실행) → PNG/SVG 변경 무시, 커밋 불필요

**해결**: 5단계(결과 보고) 이후에 git 상태 확인 → 판단 → 사용자 브리핑 → 최종 선택 후 처리하는 단계를 추가한다.

### 문제 #2: erd.mmd 수정 여부 판단 기준 미명시

**원인**: 스킬 문서에 "erd.mmd 수정 시에만 커밋" 규칙이 없음.
**해결**: 6단계에 판단 기준을 명확히 기술한다.

## 3. 스킬 업데이트 제안

### 3-1. SKILL.md 변경

**위치**: 행 89~93 (5단계: 결과 보고) 이후에 **6단계 추가**

```markdown
### 6단계: Git 후속 처리

미리보기 완료 후 git 상태를 확인하고 자동으로 판단한다.

**판단 기준:**

```bash
git diff --name-only
```

| 상황 | 판단 | 처리 |
|------|------|------|
| `erd.mmd`가 변경됨 | PNG/SVG도 커밋 대상 | 3개 파일 모두 스테이징 → 커밋 |
| `erd.mmd` 미변경 (preview만) | PNG/SVG 변경 무시 | 커밋 불필요 |

**자동 처리 절차:**

1. `git diff --name-only`로 변경 파일 목록 확인
2. `erd.mmd` 포함 여부 판단
3. 사용자에게 브리핑:

```
📊 ERD Git 처리 브리핑

변경된 파일:
  - docs/erd/erd.mmd  ← 수정됨  (또는 미변경)
  - docs/erd/erd.png  ← 재렌더링
  - docs/erd/erd.svg  ← 재렌더링

판단: [erd.mmd 수정됨 → 3개 파일 커밋 권장] 또는 [erd.mmd 미변경 → 커밋 불필요]

커밋 메시지 (안): "docs: ERD 업데이트 - {변경 내용 한 줄 요약}"

진행하시겠습니까?
  A) 커밋 진행
  B) 스테이징만 (커밋 보류)
  C) 처리 안 함
```

4. 사용자 선택에 따라 처리:
   - **A 선택**: `git add docs/erd/erd.mmd docs/erd/erd.png docs/erd/erd.svg && git commit -m "..."`
   - **B 선택**: `git add docs/erd/erd.mmd docs/erd/erd.png docs/erd/erd.svg`
   - **C 선택**: 처리 없이 종료
```

## 4. 검증 시나리오

### 시나리오 1: erd.mmd 수정 후 preview → 커밋까지 자동 처리

**목적**: erd.mmd 수정 시 PNG/SVG 포함 커밋이 자동으로 제안되는지 확인
**전제**: `docs/erd/erd.mmd` 파일이 존재하고 git 추적 중

```bash
# 단계 1: erd.mmd에 테이블 1개 추가 (수정 발생)
# 단계 2: /peach-erd 실행하여 preview
# 단계 3: 6단계에서 git diff 확인
git diff --name-only
# 기대: erd.mmd, erd.png, erd.svg 모두 표시

# 단계 4: 브리핑 출력 확인
# 기대: "erd.mmd 수정됨 → 3개 파일 커밋 권장" 메시지

# 단계 5: A 선택 시 커밋 실행
git log --oneline -1
# 기대: 커밋 메시지에 변경 내용 포함
```

**통과 기준**: 사용자가 A를 선택하면 3개 파일이 커밋됨

**통과 기준**: 커밋 불필요 판단 + C 선택이 기본값으로 제안됨
