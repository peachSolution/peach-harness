## 2. 해결 방법 / 우회 전략

### 문제 #1: AI가 스킬 파일 직접 수정

**원인**: 규칙 7이 핵심 규칙 섹션(행 299) 맨 끝에 묻혀있어 AI가 작업 도중 잊거나 무시함.
"코드 생성 금지"라는 표현이 코드(.js/.ts) 작성 금지로만 해석될 여지가 있고,
SKILL.md 수정은 "코드"가 아니라 "문서"로 인식될 수 있음.

**해결 방향**: 규칙을 스킬 최상단(페르소나/역할 바로 아래)에 경고 블록으로 배치.
"문서 생성까지만" + "스킬 파일 수정 금지"를 명시적으로 분리하여 기술.

### 문제 #2: AI가 멈춰야 할 경계선 불명확

**원인**: 스킬 워크플로우가 5단계(저장 및 안내)에서 종료됨이 명시되어 있지 않음.
"다음 에이전트를 위한 작업 지시"가 문서 섹션에도 포함되어 있어
현재 AI가 그 지시를 직접 수행해야 한다고 오해할 수 있음.

**해결 방향**: 5단계 완료 = 이 스킬의 작업 종료임을 명시.
완료 후 AI가 취해야 할 액션: "사용자에게 결과 보고 후 대기" 를 명문화.

**위치**: 스킬 제목 바로 아래 (행 1~5 사이)

**추가할 내용**:
```markdown
> ⛔ **이 스킬의 책임 범위: 문서 생성까지**
>
> - **할 것**: 피드백 문서(`docs/스킬피드백/TODO-*.md`) 생성
> - **하지 말 것**: 대상 스킬의 SKILL.md 수정, references/ 파일 변경, 코드 수정
>
> 문서를 생성하고 사용자에게 경로를 알려주면 이 스킬의 역할은 끝난다.
> 실제 스킬 반영은 **다른 세션의 다른 AI 에이전트**가 문서를 읽고 수행한다.
```

**현재**:


**변경 후**:
```
7. **스킬 파일 절대 수정 금지** — SKILL.md, references/, 기타 스킬 파일을 직접 Edit/Write 금지.
   문서(TODO-*.md)를 생성하고 사용자에게 보고하면 이 스킬의 작업은 완료된다.
   스킬 반영은 반드시 다른 세션의 에이전트가 피드백 문서를 읽고 수행한다.
8. **작업 종료 기준** — `docs/스킬피드백/TODO-*.md` 파일을 Write tool로 저장한 순간 작업 완료.
   그 이후 어떠한 파일도 수정하지 않는다.
```
