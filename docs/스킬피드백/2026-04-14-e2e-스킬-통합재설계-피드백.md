## 2. 해결 방법 / 우회 전략

### 문제 #1: peach-e2e-run의 실행-수정 단절

**원인**: run 스킬은 `./e2e.sh run` 실행 후 결과만 보고. 실패 시 에이전트가 직접 agent-browser eval로 DOM 확인 → 스크립트 수정 → 재실행을 반복해야 함.

**실제 발생한 루프** (이 세션에서):
1. 실행 → Step5 장바구니 셀렉터 실패 (updateAction 패턴 불일치)
2. DOM 확인 → 셀렉터 수정 → 재실행 → 성공
3. Step2 URL 오류 (step2 → step2_1 → step21)
4. DOM 확인 → URL 수정 → 재실행
5. validation 실패 (미사용 항목)
6. DOM 확인 → 미사용 항목 삭제 로직 추가 → 재실행
7. 포장재 기본값 자동세팅 → 기대값 수정 → 재실행 → 성공

**해결 방향**: 실행 + 자동 수정 루프를 스킬 내부에 통합

### 문제 #2: peach-e2e-convert 비사용

**원인**: Playwright codegen 녹화 스크립트보다 기존 시나리오(1-주문등록.js, 2-결제.js)의 패턴을 참조해서 직접 작성하는 것이 더 정확하고 빠름.

**실제 패턴**: 이 세션에서 시나리오 생성 시:
- `connect()` 호출 패턴 → 기존 코드에서 복사
- dialog 처리 패턴 → 2-결제.js에서 참조
- 셀렉터 패턴 → agent-browser eval로 실시간 확인

**해결 방향**: convert를 독립 스킬이 아닌, 시나리오 생성의 입력 옵션으로 흡수

### 문제 #5: 반복 루프 토큰 낭비

**원인**: 실패 → DOM 확인 → 수정이 각각 별도 도구 호출. 스킬이 이 루프를 자동화하지 않음.

**해결 방향**: 스킬 내부에서 실행 실패 시 자동으로:
1. 에러 메시지 파싱 (셀렉터 timeout, URL 오류 등)
2. agent-browser eval로 실제 DOM 조사
3. 스크립트 자동 수정
4. 재실행 (최대 3회)

## 3. 스킬 업데이트 제안

### 3-1. 신규 스킬 생성: `peach-e2e-scenario`

**peach-e2e-run과 peach-e2e-convert를 통합**하여 `peach-e2e-scenario` 신규 생성.

#### 모드

| 모드 | 트리거 | 동작 |
|------|--------|------|
| `auto` (기본) | `/peach-e2e-scenario` | 생성 + 실행 + 자동수정 |
| `create` | `/peach-e2e-scenario create` | 시나리오 생성만 |
| `run` | `/peach-e2e-scenario run` | 실행 + 자동수정 루프 |

#### 핵심 워크플로우: auto 모드

```
1. 사용자 요청 파악 ("재주문 복사 검증해줘")
2. 기존 시나리오 패턴 참조 (e2e/시나리오/ 하위 읽기)
3. 필요 시 agent-browser eval로 DOM 구조 선조사
4. 시나리오 스크립트 생성
5. ./e2e.sh run 실행
6. 실패 시:
   ├─ 에러 파싱 (셀렉터 timeout? URL 오류? validation?)
   ├─ agent-browser eval로 실제 DOM 확인
   ├─ 스크립트 자동 수정
   └─ 재실행 (최대 3회)
7. 성공 시 결과 보고
```

#### create 모드 입력 소스

| 입력 | 처리 |
|------|------|
| 자연어 요청 | 기존 시나리오 패턴 기반 생성 |
| Playwright codegen 녹화 | 변환 규칙 적용 후 생성 (기존 convert 로직) |

#### run 모드 자동수정 판단 트리

```
실행 실패
├─ locator timeout → agent-browser eval로 셀렉터 조사 → 수정
├─ waitForURL timeout → 현재 URL 확인 → URL 패턴 수정
├─ navigation timeout → AJAX submit 여부 확인 → waitForNavigation 제거
├─ dialog block → dialog handler 추가
└─ 서버 에러 (500) → 로그 확인 → 사용자에게 보고 (자동수정 불가)
```

## 4. 검증 시나리오

### 시나리오 1: auto 모드 — 자연어 요청으로 시나리오 생성+실행

**목적**: 사용자가 자연어로 요청하면 시나리오 생성부터 실행까지 자동 처리되는지 확인
**전제**: Chrome Beta CDP 실행, 로그인 완료, 기존 주문 1건 이상 존재

```
1. /peach-e2e-scenario "재주문 복사 후 옵션 초기화 검증"
2. 스킬이 기존 시나리오 패턴 참조하여 스크립트 자동 생성
3. ./e2e.sh run 실행
4. 실패 시 자동 수정 → 재실행
5. 성공 보고
```

**기대 결과**: 사용자 개입 없이 시나리오 생성 → 실행 → (실패 시 수정) → 성공
**통과 기준**: 최대 3회 자동수정 내에 전체 통과

### 시나리오 2: create 모드 — codegen 녹화 입력

**목적**: Playwright codegen 녹화 스크립트를 입력하면 CDP 시나리오로 변환되는지 확인
**전제**: codegen으로 녹화된 스크립트 준비

```
1. /peach-e2e-scenario create (녹화 스크립트 붙여넣기)
2. 변환 규칙 적용 확인 (connect(), dialog handler, exitCode 패턴)
3. 생성된 파일 확인
```

**기대 결과**: e2e/시나리오/ 하위에 변환된 파일 저장
**통과 기준**: 변환 규칙 6개 항목 모두 적용됨

### 시나리오 3: run 모드 — 자동수정 루프

**목적**: 의도적으로 잘못된 셀렉터가 있는 시나리오를 실행해 자동수정이 작동하는지 확인
**전제**: 잘못된 셀렉터(`#wrong_selector`)가 포함된 테스트 시나리오

```
1. /peach-e2e-scenario run (잘못된 시나리오)
2. 1차 실행 실패 → 에러 파싱
3. agent-browser eval로 실제 DOM 확인
4. 셀렉터 자동 수정
5. 2차 실행 → 성공
```

**기대 결과**: 1회 자동수정으로 성공
**통과 기준**: 수정된 셀렉터가 실제 DOM과 일치
