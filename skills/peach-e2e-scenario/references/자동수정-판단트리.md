# 자동수정 판단 트리

실행 실패 시 에러 메시지를 파싱하여 유형을 판별하고, 유형별 수정 전략을 적용한다.
최대 3회 자동수정 후에도 실패하면 사용자에게 에러 이력을 보고하고 중단한다.

---

## 1. 셀렉터 불일치 (locator timeout / not found)

### 에러 패턴
```
Timeout 30000ms exceeded.
waiting for locator('...')
locator.click: Error: Timeout
```

### 확인
```bash
agent-browser connect 9222

# 셀렉터 존재 여부
agent-browser eval "document.querySelector('.target') !== null"

# 유사 셀렉터 탐색 — 버튼 목록
agent-browser eval "JSON.stringify(Array.from(document.querySelectorAll('button')).map(function(b){return b.innerText}))"

# 특정 role 탐색
agent-browser eval "JSON.stringify(Array.from(document.querySelectorAll('[role=dialog] button')).map(function(b){return b.innerText}))"

# data-testid 목록
agent-browser eval "JSON.stringify(Array.from(document.querySelectorAll('[data-testid]')).map(function(e){return e.dataset.testid}))"
```

### 수정 전략
- 에러 메시지에서 실패한 셀렉터 추출
- agent-browser eval로 실제 DOM에서 올바른 셀렉터 확인
- 시나리오 파일에서 해당 셀렉터 교체

### 수정 불가 기준
- 대상 요소가 DOM에 아예 존재하지 않는 경우 (페이지 상태 문제)
- 동적 렌더링으로 타이밍에 따라 요소가 나타나지 않는 경우 → 대기 로직 추가 시도

---

## 2. URL 패턴 불일치 (waitForURL timeout)

### 에러 패턴
```
page.waitForURL: Timeout 30000ms exceeded.
waiting for navigation to "..."
```

### 확인
```bash
# 현재 실제 URL
agent-browser eval "location.href"

# URL 경로만
agent-browser eval "location.pathname"
```

### 수정 전략
- 실제 이동된 URL과 기대 URL 패턴 비교
- URL에 동적 파라미터(step2 → step2_1 → step21 등)가 있으면 와일드카드 패턴으로 수정
- 예: `waitForURL('**/step2')` → `waitForURL('**/step2*')`

### 수정 불가 기준
- 페이지 이동 자체가 발생하지 않은 경우 (서버 에러, 권한 문제)

---

## 3. dialog 자동 닫힘

### 에러 패턴
```
기록하지 못했습니다
No dialog is showing
dialog.accept: Protocol error
```

### 확인
```bash
# daemon 잔류 확인
lsof -iTCP:9222 -sTCP:ESTABLISHED -n -P

# daemon 종료
lsof -iTCP:9222 -sTCP:ESTABLISHED -n -P | awk '/node/{print $2}' | xargs kill -9 2>/dev/null
```

### 수정 전략
1. daemon 잔류 → kill -9로 종료
2. `setDialogHandler` 패턴 미적용 → `connect.js`의 `setDialogHandler(page, prevHandler, newHandler)` 패턴으로 교체
3. `page.on('dialog')` 다중 등록 → 교체 패턴으로 변경

상세: `references/daemon-잔류-대응.md`, `references/dialog-handler-패턴.md` 참조

### 수정 불가 기준
- daemon이 아닌데도 dialog가 즉시 닫히는 경우 (Chrome 자체 동작)

## 4. popup 관련 에러

### 에러 패턴
```
waitForEvent('close'): Timeout
Target page, context or browser has been closed
```

### 수정 전략
- `waitForEvent('close')` → `Promise.race` + `.catch(() => {})` 패턴 적용
- 팝업이 예상보다 빨리 닫히는 경우 → `.catch(() => {})` 추가
- `closeTarget` 옵션이 필요한 경우 추가
- `Target page … has been closed` 에러가 앞 시나리오의 CDP UA 변경 이후 발생한다면 → §8 참조

---

## 5. navigation timeout (AJAX submit)

### 에러 패턴
```
page.waitForNavigation: Timeout
waiting for navigation
```

### 확인
```bash
# AJAX 요청 여부 — 페이지가 이동하지 않고 현재 URL 유지되는지 확인
agent-browser eval "location.href"
```

### 수정 전략
- `waitForNavigation` 제거
- `waitForURL()` 또는 특정 요소 대기로 교체
- SPA는 URL 변경 없이 DOM만 변경될 수 있음 → `waitForSelector()` 또는 `waitForFunction()` 사용

## 6. 서버 에러

### 에러 패턴
```
500 Internal Server Error
net::ERR_CONNECTION_REFUSED
net::ERR_NAME_NOT_RESOLVED
```

### 수정 전략
- **자동수정 불가** — 스크립트 문제가 아닌 서버/네트워크 문제
- 에러 메시지를 사용자에게 보고하고 중단

---

## 7. 분류 불가 에러

### 수정 전략
- 에러 메시지 + agent-browser eval로 현재 페이지 상태(URL, DOM 스냅샷) 수집
- 수집 결과를 사용자에게 보고
- `peach-e2e-browse` 스킬로 수동 디버깅 안내

## 7. AJAX 결과 로딩 지연 (고정 대기 오탐)

### 에러 패턴
```
✅ 처럼 보이지만 실제로는 "검색결과가 없습니다" 텍스트를 읽음
이력/목록이 있어야 하는데 항상 빈 결과 반환
```

### 원인
`waitForTimeout(N)` 고정 대기 중 AJAX 응답이 아직 도착하지 않은 상태에서 텍스트를 읽음.

### 수정 전략
특정 "빈 결과" 텍스트가 사라질 때까지 폴링 (최대 5초, 500ms 간격).

```javascript
async function waitForAjaxResult(frame, elementId, emptyText = '검색결과가 없습니다', maxTries = 10) {
    for (let i = 0; i < maxTries; i++) {
        await frame.waitForTimeout(500);
        const text = await frame.evaluate((id) => {
            const el = document.getElementById(id);
            return el ? el.innerText : '';
        }, elementId);
        if (text && !text.includes(emptyText)) return text;
    }
    return await frame.evaluate((id) => {
        const el = document.getElementById(id);
        return el ? el.innerText : '';
    }, elementId);
}

// 사용
const histText = await waitForAjaxResult(iframe, 'hist_list_html');
```

### 수정 불가 기준
- 데이터 자체가 없는 경우 (서버 조건 문제 → `validation-통과-패턴.md` §7 참조)

## 8. CDP UA 변경 후 컨텍스트 닫힘

### 에러 패턴
```
page.goto: Target page, context or browser has been closed
```
앞 시나리오에서 모바일 UA를 설정한 뒤 다음 시나리오 실행 시 발생.

### 원인
CDP `Network.setUserAgentOverride`로 UA를 변경한 탭이 다음 연결 시 컨텍스트 불일치.

### 수정 전략
1. **재실행 1회**로 자동 복구됨 (`connect()`가 유효한 탭을 재선택)
2. **예방**: UA 변경 시나리오 `finally`에서 UA 복원

```javascript
} finally {
    try {
        const client = await page.context().newCDPSession(page);
        await client.send('Network.setUserAgentOverride', { userAgent: '' }); // 기본값 복원
    } catch (_) {}
    process.exit(process.exitCode || 0);
}
```

### 수정 불가 기준
- Chrome CDP 자체 문제 → Chrome Beta 재시작 필요
