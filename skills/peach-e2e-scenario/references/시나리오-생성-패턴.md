# 시나리오 생성 패턴

자연어 요청에서 E2E 시나리오를 처음부터 생성할 때 따르는 가이드.
Playwright codegen 변환이 아닌, **기존 시나리오 코드를 참조하여 직접 작성**하는 방식.

---

## 1. 기존 시나리오 탐색

시나리오 작성 전 반드시 기존 코드를 읽어 프로젝트의 패턴을 파악한다.

```bash
# 시나리오 목록 확인
ls e2e/시나리오/**/*.js

# 유사 시나리오 찾기 (예: 주문등록 관련)
ls e2e/시나리오/*주문* e2e/시나리오/**/*주문* 2>/dev/null
```

참조할 것:
- `connect()` 호출 패턴 → 기존 코드에서 복사
- dialog 처리 패턴 → 기존 코드에서 참조
- 셀렉터 패턴 → 프로젝트별 DOM 구조에 맞춤
- 단계 분리 패턴 → 이모지 로그 + 단계 번호

## 2. DOM 선조사

기존 시나리오만으로 셀렉터를 판단하기 어려우면, agent-browser eval로 실시간 DOM을 확인한다.
단, Google 로그인/OAuth/관리자 콘솔/결제/기존 프로필 세션 유지 작업은
탭 확인과 사용자 승인 전까지 분석만 수행하고 조작하지 않는다.

```bash
# 먼저 사용자가 작업 탭을 확인
cd e2e && ./e2e.sh status

# 사용자가 탭을 지정한 뒤에만 CDP 연결 및 DOM 조사 진행
agent-browser connect 9222

# 페이지 URL 구조 파악
agent-browser eval "location.pathname"

# 주요 UI 요소 탐색
agent-browser eval "JSON.stringify(Array.from(document.querySelectorAll('button')).map(function(b){return b.innerText}))"

# form 필드 목록
agent-browser eval "JSON.stringify(Array.from(document.querySelectorAll('input,select,textarea')).map(function(e){return {tag:e.tagName,name:e.name,id:e.id,type:e.type}}))"

# 테이블 구조
agent-browser eval "document.querySelectorAll('table').length + '개 테이블'"
```

> 탭 번호 미확인 상태에서는 위 `eval` 예시를 실행하지 않는다.
> 민감 세션 작업은 탭 번호를 받아도 사용자 승인 전까지 DOM 조사 대신 상태 설명만 수행한다.

## 3. 시나리오 기본 구조

```javascript
/**
 * {시나리오 설명}
 *
 * 사전조건:
 *   - Chrome Beta CDP 모드 실행
 *   - 브라우저 로그인 완료
 *   - {필요한 페이지} 접속
 */
const { connect } = require('../../lib/connect');

(async () => {
  console.log('🚀 [{시나리오명}] 시작');
  let exitCode = 0;
  const { page, origin } = await connect();
  console.log(`📍 현재 페이지: ${page.url()}`);

  // 페이지 체크 — 안전한 기본값은 자동 이동 금지
  if (!page.url().includes('/expected/path')) {
    console.log('👀 현재 탭이 예상 경로가 아닙니다.');
    console.log(`📍 현재 URL: ${page.url()}`);
    throw new Error('예상 경로가 아님 — 사용자 확인 후 다시 실행하세요.');
    // 비민감 세션 + 사용자 실행 승인 확보 시에만 아래 자동 이동 사용:
    // await page.goto(`${origin}/expected/path`);
    // await page.waitForURL('**/expected/path');
  }

  try {
    // === 단계별 시나리오 ===
    console.log('\n📋 [1] 단계 설명');
    // 조작 코드
    console.log('✅ 완료');

    console.log('\n📋 [2] 다음 단계');
    // 조작 코드
    console.log('✅ 완료');

    console.log('\n✨ [{시나리오명}] 완료!');
    console.log('👀 화면을 확인하세요.');
  } catch (e) {
    exitCode = 1;
    console.error('\n❌ 에러:', e.message);
  } finally {
    process.exit(exitCode);
  }
})();
```

## 4. dialog handler 패턴 (필요시만)

native dialog(`alert`, `confirm`)가 발생하는 시나리오에서만 추가한다.

```javascript
const { connect, setDialogHandler } = require('../../lib/connect');

(async () => {
  let exitCode = 0;
  const { page, origin, defaultDialogHandler } = await connect();

  // dialog handler 교체
  const myHandler = setDialogHandler(page, defaultDialogHandler, async (dialog) => {
    console.log(`📢 다이얼로그: ${dialog.message()}`);
    try { await dialog.accept(); } catch (_) {}
  });

  try {
    // 시나리오 코드
  } catch (e) {
    exitCode = 1;
    console.error('\n❌ 에러:', e.message);
  } finally {
    // handler 복원 필수
    setDialogHandler(page, myHandler, defaultDialogHandler);
    process.exit(exitCode);
  }
})();
```

## 5. 셀렉터 우선순위

1. `getByTestId()` — data-testid (가장 안정적)
2. `getByRole()` — semantic role
3. `getByLabel()`, `getByPlaceholder()` — 입력 필드
4. `getByText()` — 텍스트 매칭
5. CSS: `#id`, `[data-*]` — 위 방법이 안 될 때

> 해시 클래스(예: `.css-1a2b3c`) 사용 금지 — 빌드마다 변경됨.

## 6. 대기 전략

| 상황 | 패턴 |
|------|------|
| SPA 라우트 이동 | `waitForURL('**/path')` |
| MPA 풀 리로드 | `waitForLoadState('networkidle')` |
| 요소 나타남 대기 | `waitForSelector('.target')` 또는 `locator.waitFor()` |
| 데이터 로딩 대기 | `waitForFunction(() => condition)` |
| AJAX 완료 대기 | `waitForResponse(url => url.includes('/api/'))` |

> `waitForTimeout()` 사용 최소화 — 이벤트 기반 대기로 교체.

## 8. 다단계 시나리오 번호 설계 규칙

의존 관계가 있는 시나리오는 **데이터 흐름 순서**로 번호를 부여한다.

```
원칙: 데이터 생성 → 생성 직후 검증 → 수정/활용 → 독립 검증
번호:     01             02              03             04
```

**잘못된 예 (의존성 역전)**:
```
01-수정-저장-검증.js   ← admin PK 필요 (state.json 의존)
02-신규등록-검증.js   ← admin PK 생성 (state.json 저장)
```

**올바른 예**:
```
01-신규등록-검증.js   ← PK 생성 + state.json 저장
02-변경이력-검증.js   ← 01의 PK 재활용
03-수정-저장-검증.js  ← 01의 PK 재활용
04-목록필터-검증.js   ← 독립 시나리오
```

번호가 의존성 역전 상태로 생성되었다면 `git mv`로 rename한다.

## 9. 다단계 시나리오 PK 공유 — state.json 패턴

단위 시나리오는 독립 node 프로세스 → 메모리 공유 불가.
`e2e/.tmp/e2e_{기능명}_state.json` 파일로 PK/이름을 전달한다.

**저장 (PK 생성 시나리오)**:
```javascript
const path = require('path');
const fs = require('fs');
const STATE_FILE = path.join(__dirname, '../../../.tmp/e2e_{기능명}_state.json');
let state = {};
try { state = JSON.parse(fs.readFileSync(STATE_FILE, 'utf8')); } catch (_) {}
state.target = { pk: N, name: TEST_NAME };
fs.mkdirSync(path.dirname(STATE_FILE), { recursive: true });
fs.writeFileSync(STATE_FILE, JSON.stringify(state, null, 2));
console.log(`💾 상태 저장: pk=${N}`);
```

**로드 (이후 시나리오)**:
```javascript
const path = require('path');
const fs = require('fs');
const STATE_FILE = path.join(__dirname, '../../../.tmp/e2e_{기능명}_state.json');
let state = {};
try { state = JSON.parse(fs.readFileSync(STATE_FILE, 'utf8')); } catch (_) {}
const PK = state.target?.pk
    ? String(state.target.pk)
    : (process.env.E2E_PK || null);  // 단독 실행 fallback
if (!PK) throw new Error('state.json 없음. 01-등록 시나리오를 먼저 실행하세요.');
```

**필수**: `e2e/.gitignore`에 `.tmp/` 추가.

## 10. 테스트 데이터 이력 유지 — finally DELETE 금지

`finally` 블록에서 테스트 데이터를 DELETE하면 이후 시나리오의 교차 검증(관리자 목록 노출 확인 등)에서
데이터가 사라져 오탐이 발생한다.

```javascript
// ❌ 하지 말 것 — fetch/API 호출로 삭제하는 경우도 동일
} finally {
    // await page.evaluate(() => fetch('/module/deleteSubmit', ...));  ← 금지
    process.exit(process.exitCode || 0);
}

// ✅ 이력 유지 + 로그 출력
} finally {
    console.log(`\n📌 등록 이력 유지: pk=${N} (${TEST_NAME})`);
    process.exit(process.exitCode || 0);
}
```

운영 데이터 정리는 별도 정리 스크립트 또는 운영자 수동 처리로 진행한다.
