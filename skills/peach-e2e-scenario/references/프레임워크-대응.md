# 프레임워크별 E2E 대응 가이드

레거시(PHP/jQuery)와 모던 프레임워크(React/Vue/Next.js)의 E2E 패턴 차이점.

---

## 모달 처리

### 레거시 (PHP + jQuery UI Dialog + iframe)

```javascript
// iframe 내부 접근
const frame = page.frameLocator('#ui-id-1');
await frame.locator('a', { hasText: '실행' }).waitFor({ timeout: 10000 });
await frame.locator('a', { hasText: '실행' }).click();

// 오버레이 강제 제거
await page.evaluate(() => {
  document.querySelectorAll('.ui-widget-overlay').forEach(el => el.remove());
  document.querySelectorAll('.ui-dialog').forEach(el => el.style.display = 'none');
});
```

### React (MUI Dialog / Radix UI / Headless UI)

```javascript
// iframe 없이 DOM에 직접 렌더링 (React Portal)
// role="dialog" 로 접근
const modal = page.getByRole('dialog');
await modal.waitFor({ timeout: 10000 });
await modal.getByRole('button', { name: '실행' }).click();

// 오버레이 강제 제거 (필요시)
await page.evaluate(() => {
  document.querySelectorAll('.MuiBackdrop-root').forEach(el => el.remove());
  // Radix UI
  document.querySelectorAll('[data-radix-popper-content-wrapper]').forEach(el => el.remove());
});
```

### Vue (Nuxt UI / Vuetify / PrimeVue)

```javascript
// Nuxt UI: role="dialog" 동일
const modal = page.getByRole('dialog');
await modal.getByRole('button', { name: '확인' }).click();

// Vuetify: .v-overlay__content 내부
const modal = page.locator('.v-overlay__content');
await modal.getByRole('button', { name: '확인' }).click();
```

### SPA 공통 주의사항

- iframe 모달은 거의 없음 → `frameLocator` 불필요
- `role="dialog"` + `aria-modal="true"` 패턴이 표준
- 애니메이션 대기: `waitFor({ state: 'visible' })` 또는 짧은 `waitForTimeout(300)`

## 팝업 (window.open) 처리

### 레거시 PHP

```javascript
// window.open 새창 → popup 이벤트
const popupPromise = page.waitForEvent('popup');
await page.locator('a', { hasText: '관리' }).click();
const popup = await popupPromise;

popup.on('dialog', async (dialog) => {
  try { await dialog.accept(); } catch (_) {}
});
await popup.waitForLoadState('networkidle');
await popup.locator('a', { hasText: '삭제' }).click();
await popup.waitForEvent('close', { timeout: 5000 }).catch(() => {});
```

### React / Vue / Next.js (SPA)

- `window.open` 새창은 거의 사용하지 않음
- 대신 라우터 기반 페이지 이동 또는 모달 사용
- 새창이 필요한 경우(외부 링크 등)는 동일하게 `popup` 이벤트 사용

```javascript
// SPA에서 새창이 열리는 경우 (드물지만 동일 패턴)
const popupPromise = page.waitForEvent('popup');
await page.getByRole('link', { name: '미리보기' }).click();
const popup = await popupPromise;
await popup.waitForLoadState('networkidle');
```

## 셀렉터 전략

### 레거시 (PHP + jQuery)

```javascript
// id/class 기반 (안정적)
await page.locator('#customer_name').fill('홍길동');
await page.locator('span.button a', { hasText: '검색' }).click();
await page.locator('a.hoya_button', { hasText: '다음으로' }).click();

// 테이블 내 링크 (사이드바와 구분 필수)
await page.locator('table tbody td').getByRole('link', { name: '관리', exact: true }).first().click();
```

### React / Vue / Next.js

```javascript
// data-testid 우선 (가장 안정적)
await page.getByTestId('submit-button').click();

// aria role/label (시맨틱)
await page.getByRole('button', { name: '저장' }).click();
await page.getByLabel('이름').fill('홍길동');
await page.getByPlaceholder('검색어 입력').fill('검색어');

// 컴포넌트 클래스는 빌드마다 변경될 수 있어 불안정
// ❌ page.locator('.sc-bcXHqe') — CSS-in-JS 해시 클래스
// ✅ page.getByRole() 또는 getByTestId()
```

## 오버레이 / 백드롭 제거

| 프레임워크 | 오버레이 셀렉터 |
|-----------|----------------|
| jQuery UI | `.ui-widget-overlay` |
| MUI (React) | `.MuiBackdrop-root` |
| Vuetify | `.v-overlay__scrim` |
| Nuxt UI | `.fixed.inset-0` (Tailwind 기반) |
| Radix UI | `[data-radix-scroll-lock-input]` 근처 |
| 범용 | `[role="presentation"]`, `[aria-hidden="true"]` |

```javascript
// 범용 오버레이 제거 패턴
await page.evaluate(() => {
  // 각 프레임워크에 맞는 셀렉터로 교체
  document.querySelectorAll('.ui-widget-overlay, .MuiBackdrop-root, .v-overlay__scrim')
    .forEach(el => el.remove());
});
```

## 페이지 이동 대기

### 레거시 (서버사이드 렌더링 / 풀페이지 리로드)

```javascript
await page.locator('a', { hasText: '다음으로' }).click();
await page.waitForLoadState('networkidle');
```

### React / Vue / Next.js (SPA 클라이언트 라우팅)

```javascript
// networkidle 대신 URL 변경 또는 특정 요소 대기
await page.getByRole('button', { name: '다음' }).click();
await page.waitForURL('**/step2');
// 또는
await page.locator('[data-testid="step2-content"]').waitFor();
```

## Flutter 웹앱

Flutter 웹앱은 Canvas 렌더링이므로 Playwright codegen 녹화가 정상 동작하지 않는다.
**Flutter 사이트의 E2E 자동화는 codegen → convert 파이프라인의 대상이 아니다.**

Flutter 사이트 자동화가 필요하면:
- `peach-e2e-browse` 스킬의 **API 직접 호출** 전략을 사용한다
- `references/Flutter-웹앱-패턴.md` (peach-e2e-browse 스킬 내) 참조

판별법: `eval "!!document.querySelector('flt-semantics-placeholder')"` → true이면 Flutter

## 핵심 차이 요약

| 항목 | 레거시 (PHP/jQuery) | React/Vue/Next.js |
|------|------------------|-------------------|
| 모달 구조 | iframe + jQuery UI Dialog | React Portal / Teleport (DOM 직접) |
| 모달 접근 | `page.frameLocator()` | `page.getByRole('dialog')` |
| 버튼 구조 | `<span class="button"><a>` | `<button>` 또는 컴포넌트 |
| 셀렉터 기반 | id/class/텍스트 | role/testid/label |
| 페이지 이동 | 풀 리로드 → `networkidle` | 클라이언트 라우팅 → `waitForURL` |
| 팝업 | `window.open` 빈번 | 거의 없음, 모달로 대체 |
| 오버레이 | `.ui-widget-overlay` | 프레임워크별 상이 |
