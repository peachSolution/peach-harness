# 코드 수정 위임 정책

`peach-team-e2e`는 사용자 경험 검증 스킬이다. 본 프로젝트 코드 수정은 직접 하지 않고, 안전 조건을 만족할 때 `peach-team-dev`로 위임한다.

## 자동 위임 가능

다음 조건을 모두 만족하면 사용자 확인 없이 `peach-team-dev`에 수정 프롬프트를 넘긴다.

- Spec 또는 ui-proto에 명확한 근거가 있다.
- 수정 대상 레이어가 backend/store/ui 중 하나로 분명하다.
- DB 구조, 공통 모듈, 권한 정책을 변경하지 않는다.
- 기존 Must Follow를 침범하지 않는다.
- 수정 후 TDD/lint/build/E2E 재실행이 가능하다.

## 사용자 확인 필요

다음 중 하나라도 해당하면 자동 위임하지 않는다.

- Spec과 ui-proto가 충돌한다.
- PRD에는 있으나 Spec/ui-proto에 없는 요구다.
- 권한, 데이터 소유 범위, 상태 전이 정책 판단이 필요하다.
- DB schema, 공통 상수, 공통 컴포넌트 변경이 필요하다.
- 수정 레이어가 불명확하다.
- 반복 실패로 자동 보완 신뢰도가 낮다.

## 위임 프롬프트 형식

```markdown
# E2E 미스매치 수정 위임

## 분류
- [Spec 비즈니스 규칙 위반 / ui-proto 화면 흐름 불일치]

## 근거
- Spec TEST_ID: [...]
- ui-proto 화면: [...]
- 실제 동작: [...]
- 기대 동작: [...]

## 수정 범위
- 레이어: backend/store/ui
- 금지: DB 구조 변경, 공통 모듈 변경, 권한 정책 임의 변경

## 재검증
- TDD/lint/build
- 실패한 E2E suite 재실행
```

위임 후 `peach-team-e2e`는 수정 결과를 받아 실패한 suite만 재실행한다.
