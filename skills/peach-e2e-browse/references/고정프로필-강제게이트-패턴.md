# 고정 프로필 강제 게이트 패턴

Chrome Beta의 `$HOME/.chrome-beta-e2e-profile`에 로그인 세션이 유지된 상태에서 작업할 때 적용하는 보수 규칙.

## 핵심 목적

- Google 계정, OAuth, 사내 SSO, 2차 인증을 **매번 새로 하지 않도록** 고정 프로필에 세션을 유지한다.
- AI는 로그인 자체를 대신하지 않고, **사용자가 이미 인증을 끝낸 같은 프로필 세션을 이어받아** 작업한다.
- 인증이 다시 필요해지면 AI는 사용자가 Chrome Beta에서 직접 로그인/인증을 완료하도록 안내한다.

## 강제 중단이 필요한 작업

아래 중 하나라도 해당하면 브라우저 조작 전에 반드시 멈춘다.

- Google 로그인, Google Workspace, Gmail, Google OAuth 설정
- 관리자 콘솔, 결제, 운영 데이터 조회/수정 화면
- 사용자가 직접 로그인한 기존 세션을 그대로 이어받아야 하는 작업
- `--user-data-dir=$HOME/.chrome-beta-e2e-profile` 유지가 핵심인 작업

## 허용되는 행동

- `curl -s http://127.0.0.1:9222/json/version`
- `agent-browser connect 9222`
- `agent-browser tab list`
- 현재 상태 설명, 탭 목록 보고, 필요한 확인 질문
- "이 탭에서 먼저 로그인/인증을 완료해 주세요. 완료되면 이어서 확인하겠습니다." 같은 안내

## 금지되는 행동

사용자가 탭 번호를 명시하기 전:

- `agent-browser open`
- `agent-browser tab N`
- `agent-browser tab new`
- `agent-browser eval`
- `agent-browser click`
- `agent-browser fill`
- `agent-browser press`

탭 번호를 받았더라도 민감 세션 작업에서 추가 승인 전:

- 설정 변경, 버튼 클릭, 입력, 제출, 페이지 이동
- 새 탭 생성, 다른 탭 전환, 직접 URL 이동

## 우회 금지 규칙

`agent-browser`가 비정상 응답이면 다음 우회는 금지다.

- `open -a "Google Chrome Beta" "https://..."`
- `open -a "Google Chrome" "https://..."`
- 다른 브라우저 실행
- 다른 `--user-data-dir` 경로 사용

이 경우에는 실패 사실, 현재 연결 상태, 다음에 필요한 사용자 확인 사항만 보고한다.

## 기본 응답 패턴

1. `connect 9222`
2. `tab list`
3. 탭 목록과 현재 상태 보고
4. "몇 번 탭에서 작업할까요?" 확인
5. 민감 세션이면 "분석까지만 진행할지, 실제 조작까지 승인하는지" 추가 확인
6. 로그인/인증이 필요한 상태면 "Chrome Beta 고정 프로필에서 직접 로그인/인증을 완료해 달라"고 안내
