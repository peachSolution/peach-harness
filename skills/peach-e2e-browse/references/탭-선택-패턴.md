# 탭 선택 패턴

## 문제: `--tab N` 인덱스가 흔들리는 이유

`--tab N`은 CDP `/json` 엔드포인트가 반환하는 배열 순서에 의존한다.
이 순서는 탭 생성/닫기/이동에 따라 변할 수 있어, 같은 N이 다른 탭을 가리킬 수 있다.

- 탭을 닫으면 인덱스가 밀린다
- 새 탭을 열면 인덱스가 추가된다
- 브라우저 내부적으로 탭 순서가 재배열될 수 있다

## 강제 게이트

`tab list`는 단순 탐색이 아니라 **실행 게이트**다.

- 사용자 탭 번호 확인 전에는 `open`, `tab`, `click`, `fill`, `press`, `eval` 실행 금지
- Google 로그인, OAuth, 관리자 콘솔, 결제, 기존 로그인 세션 유지 작업은
  탭 번호를 확인해도 사용자 실행 승인 전까지 분석만 수행
- `agent-browser`가 비정상이면 탭 우회 조작 대신 상태를 보고하고 대기

## targetId 조회 방법

```bash
# CDP에서 targetId 조회
curl -s http://127.0.0.1:9222/json | jq -r '.[] | select(.type=="page" and (.url|startswith("chrome")|not)) | [.id,.title,.url] | @tsv'
```

출력 예:
```
1A2B3C4D5E6F	NAVER	https://www.naver.com/
7G8H9I0J1K2L	Google	https://www.google.com/
```

## 실행 방법

### 일반 실행: `--tab N` (빠른 실행용)

```bash
cd e2e && ./e2e.sh run --tab 0 시나리오/대상.js
```

탭 목록이 안정적이고, 빠르게 실행할 때 사용한다.
단, 사용자가 `tab list` 결과에서 탭을 명시적으로 골라준 뒤에만 사용한다.

### 디버깅/재현/장애분석: `E2E_TAB_ID` (targetId 고정)

```bash
cd e2e && E2E_TAB_ID=1A2B3C4D5E6F ./e2e.sh run 시나리오/대상.js
```

특정 URL의 탭을 정확히 지정해야 할 때 사용한다. 인덱스 드리프트가 발생하지 않는다.

## 선택 기준

| 상황 | 권장 방식 | 이유 |
|------|----------|------|
| 일반 시나리오 실행 | `--tab N` | 빠르고 간편 |
| 디버깅/재현 | `E2E_TAB_ID` | 탭 인덱스 드리프트 방지 |
| 장애 분석 | `E2E_TAB_ID` | 정확한 탭 지정 필수 |
| CI/자동화 | `E2E_TAB_ID` | 환경 변화에 안정적 |
