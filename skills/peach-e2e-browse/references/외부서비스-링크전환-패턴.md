# 외부 서비스 링크 전환 패턴

## 문제: `click()` 한 줄로 끝내면 안 되는 이유

Google→Gmail, 로그인 페이지→OAuth 등 외부 서비스 전환 시 `click()` 한 줄만으로는 신뢰할 수 없다.

- 리다이렉트 체인이 길어 타임아웃 발생
- 팝업 차단, 쿠키 동의 등 중간 단계에서 멈춤
- 네트워크 상태에 따라 로딩 시간 편차가 큼
- `click()` 성공 ≠ 목표 URL 도달 성공

## 권장 패턴: `load + 2초 지연 + 직접 이동 fallback`

```javascript
// 1. 클릭 시도
await page.getByRole('link', { name: 'Gmail' }).click();

// 2. load 상태 대기
try {
  await page.waitForLoadState('load', { timeout: 10000 });
} catch (_) {
  // 타임아웃 허용
}

// 3. 지연 (리다이렉트 체인 완료 대기)
await page.waitForTimeout(2000);

// 4. 목표 URL 도달 확인 → 실패 시 직접 이동
if (!page.url().includes('mail.google.com')) {
  console.log('⚠️ 링크 전환 실패, 직접 이동합니다.');
  await page.goto('https://mail.google.com');
  await page.waitForLoadState('load', { timeout: 15000 });
}

// 5. 최종 검증: "클릭 성공"이 아닌 "목표 URL 도달 성공" 기준
console.log(`📍 현재 URL: ${page.url()}`);
if (!page.url().includes('mail.google.com')) {
  throw new Error('Gmail 진입 실패');
}
```

## 핵심 원칙

- **검증 기준은 "클릭 성공"이 아닌 "목표 URL 도달 성공"**
- 클릭 후 반드시 URL을 확인하고, 미도달 시 `page.goto()`로 직접 이동
- 외부 서비스는 `waitForLoadState('networkidle')` 대신 `load + 지연` 조합 사용 (외부 서비스의 지속적 네트워크 요청이 networkidle을 방해)
