## 작성 규칙

1. **Step 번호는 실행 순서** — 의존성 순서대로 배치
2. **실행 필드**: `시나리오/` 기준 상대경로. `e2e.sh run`에 전달되는 경로와 동일
3. **검증 필드**: 자연어로 기술. AI가 agent-browser eval로 확인할 수 있는 수준의 구체성 필요
   - 좋은 예: "URL이 `/order/edit/`을 포함하는가"
   - 나쁜 예: "페이지가 정상인가"
4. **전달 데이터**: 다음 Step에서 사용할 데이터만 명시. 추출 방법도 기술
5. **사전 주입**: 환경변수명은 `E2E_` 접두어. 단위 시나리오에서 `process.env.E2E_*`로 접근
6. **결과 검증 Step**: 실행 파일 없이 AI가 직접 확인하는 단계. DB/코드 검증에 사용
7. **실행 이력**: git log에서 확인. suite md 파일은 시나리오 정의만 포함하며 실행 후 변경되지 않는다.

## 네이밍 규칙

- 파일명: `{업무흐름}-{핵심동작}.md` (kebab-case, 한글 허용)
- 예시:
  - `주문-결제-검증.md`
  - `신규주문-전체흐름.md`
  - `회원가입-로그인-검증.md`
  - `재고입고-재고반영-검증.md`
